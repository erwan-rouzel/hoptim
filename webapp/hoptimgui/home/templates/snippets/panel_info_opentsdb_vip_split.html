Information

The VIP algorithm isolates the VIP metrics which have been defined in HOPtim YAML settings file. This means these metrics will then occupy one region only which will allow balancing the VIP metric equally when doing the balancing.

Information

The maximum number of Regions per Region Server is computed with the formula:

$$RegionServerMemory * MemStoreUpperLimit \over MemstoreFlushSize * NumberColumnFamilies$$

This algorithm does a split which is optimized for OpenTSDB by splitting only at the beginning of a metric. This prevents metrics from being broken between several regions.
